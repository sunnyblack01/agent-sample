## 1. Executive Summary

7개의 전문 에이전트가 고등학생 수행평가 보고서 작성의 전 과정(주제 발굴 → 문헌 조사 → 실험 설계 → 보고서 작성 → 어투 검토 → 면접 대비)을 분담하는 **인간-AI 협업 시스템**이다. 최종 결과물은 학생 본인의 보고서이며, 시스템은 뼈대와 가이드를 제공하는 역할에 머문다.

### 1.1 핵심 수치

| 항목 | 수치 |
|------|------|
| 전문 에이전트 수 | 7개 + Orchestrator 1개 |
| 커버 과목 | 화학, 물리, 수학, 생물 등 전 과목 |
| 보고서 작성 시간 목표 | 기존 대비 **50% 이상 단축** |
| AI 비감지 목표 | 최종본 기준 AI스러운 표현 **0건** |
| 대상 교육과정 | 2022 개정 교육과정 |

## 2. 문제 정의 (Problem Statement)

### 2.1 현재 상황

고등학교 수행평가는 매 과목, 매 학기 반복적으로 보고서를 요구한다. 학생은 과목마다 **주제 발굴 → 문헌 조사 → 실험 설계 → 보고서 작성**이라는 동일한 프로세스를 처음부터 반복해야 하며, 이 과정에서 상당한 시간과 에너지가 소모된다.

### 2.2 Before / After 테이블

| 구분 | Before (현재) | After (시스템 도입 후) |
|------|--------------|----------------------|
| 주제 발굴 | 매 과목마다 처음부터 탐색 | ✅ Topic Scout가 AI 진로 연계 주제 5개+ 자동 제안 |
| 문헌 조사 | 학생이 직접 검색·정리 | ✅ Research Agent가 실재 문헌 조사·정리 |
| 실험 설계 | 학생 독자 설계, 변인 누락 위험 | ✅ Experiment Designer가 가설·변인·절차 설계 |
| 보고서 작성 | 매번 백지에서 시작 | ✅ Report Writer가 뼈대 + 개인화 포인트 제공 |
| AI 생성 티 | 관리 불가 | ✅ Voice Coach가 AI스러운 표현 검출·수정 가이드 |
| 진로 키워드 | 과목마다 파편화 | ✅ Orchestrator가 과목 간 키워드 일관성 추적 |
| 면접 준비 | 별도 준비 필요 | ✅ Interview Prep이 예상 질문·답변 전략 생성 |

### 2.3 핵심 문제

| # | 문제 | 설명 |
|---|------|------|
| P1 | **반복적 작업 부담** | 매 과목 수행평가마다 주제 발굴, 실험 설계, 문헌 조사, 보고서 작성을 처음부터 반복해야 한다 |
| P2 | **진로 연결의 일관성 부족** | 여러 과목 보고서에서 AI 진로와의 연결을 일관되게 유지하기 어렵다. 과목마다 따로 작성하다 보면 키워드와 서사가 파편화된다 |
| P3 | **AI 생성 티 제약** | AI 도구를 활용하더라도 결과물이 AI가 작성한 것처럼 보이면 안 된다. 자연스러운 고등학생 어투가 필수적이다 |
| P4 | **교육과정 범위 준수** | 보고서 내용이 해당 과목의 교육과정(2022 개정) 범위를 벗어나면 평가에서 감점 요인이 된다 |
| P5 | **품질과 효율의 트레이드오프** | 시간을 절약하면서도 면접과 학교 프로그램 지원에 활용할 수 있는 고품질 보고서를 만들어야 한다 |

### 2.4 해결 방향

7개의 전문 에이전트가 보고서 작성의 각 단계를 분담하되, 학생이 각 단계에서 확인하고 개인화할 수 있는 **인간-AI 협업 구조**를 만든다. 최종 결과물은 학생 본인의 보고서이며, 시스템은 뼈대와 가이드를 제공하는 역할에 머문다.

## 3. 타겟 사용자 페르소나 (User Persona)

### 3.1 페르소나: 민수 (가명)

| 항목 | 내용 |
|------|------|
| **학년** | 고등학교 2학년 |
| **희망 진로** | AI/인공지능 학과 (대학 진학) |
| **과학 배경** | 물리 역학 기본 지식 보유, 비역학 분야 미학습 |
| **수행평가 빈도** | 매 학기 4~6개 과목에서 보고서 형태의 수행평가 수행 |
| **핵심 진로 키워드** | 데이터 기반 문제 해결, 패턴 인식, 예측, 분류, 최적화 |
| **진로 서사** | 모든 과목에서 AI적 사고방식(데이터 수집 → 패턴 발견 → 예측/분류)을 일관되게 보여주고자 함 |

### 3.2 주요 니즈

| # | 니즈 | 설명 |
|---|------|------|
| N1 | **시간 절약** | 주제 발굴과 문헌 조사에 드는 시간을 줄이고 싶다 |
| N2 | **진로 일관성** | 화학, 물리, 수학, 생물 등 여러 과목 보고서에서 AI 진로 연결이 자연스럽고 일관되게 이어지길 원한다 |
| N3 | **면접 대비** | 보고서 내용이 추후 면접 질문에 대비할 수 있는 깊이를 가져야 한다 |
| N4 | **프로그램 지원** | 학교 내외 프로그램/대회 지원서에도 활용할 수 있는 수준이어야 한다 |
| N5 | **자연스러운 어투** | 최종 보고서가 본인이 쓴 것처럼 자연스러워야 한다 |

### 3.3 사용 환경

- 학교 실험실 장비로 실행 가능한 실험만 가능
- 2022 개정 교육과정 범위 내에서 작성
- 참고 문헌은 직접 확인 가능한 실제 자료여야 함

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 기능 요구사항 총괄 테이블

| FR# | 기능명 | 담당 에이전트 | 입력 | 출력 | 제약 |
|-----|--------|--------------|------|------|------|
| FR1 | 주제 제안 | Topic Scout (01) | 과목명, 학기/단원 범위, 실험 여부 | 주제 5개+ (AI 연결 포인트, 난이도, 예상 소요 시간) | 교육과정 범위 준수, 이전 키워드 중복/연계 고려 |
| FR2 | 참고문헌 및 자료 조사 | Research Agent (02) | 선택된 주제, 과목 범위 | 참고문헌 목록(서지정보), 이론적 배경 요약, 핵심 개념 정리 | ⛔ 가짜 문헌 절대 금지. 실재 확인 가능 자료만 |
| FR3 | 실험 설계 | Experiment Designer (03) | 주제, 이론 배경, 장비/재료 정보 | 탐구 질문, 가설, 변인 설정, 준비물, 실험 절차, 안전 유의사항 | 학교 실험실 장비로 실행 가능. 위험 시약/고가 장비 지양 |
| FR4 | 보고서 뼈대 작성 | Report Writer (04) | 주제, 조사 자료, 실험 설계, 실험 결과(학생 입력) | 보고서 초안 (동기, 이론, 방법, 결과, 결론, 참고문헌) | `[본인 경험으로 채우기]` 포인트 명시. AI 연결 1~2문장 |
| FR5 | AI스러움 검토 | Voice Coach (05) | 보고서 초안 | AI스러운 표현 목록, 수정 제안, 전체 어투 평가 | 과도한 정돈, 접속부사 남용, 부적절한 어휘, 감정 없는 서술 검출 |
| FR6 | 주제 난이도/임팩트 조율 | Topic Advisor (06) | 제안된 주제 목록 또는 선택된 주제 | 난이도 평가, 임팩트 분석, 주제 변형/심화/축소 제안 | 학생이 추가 상담을 원할 때 활용 |
| FR7 | 면접 예상 질문 및 답변 전략 | Interview Prep (07) | 완성된 보고서, 지원 프로그램/대회 정보 | 예상 질문 목록, 답변 전략, 꼬리 질문 대비 포인트 | 학생 실제 경험 기반 답변이어야 함 |
| FR8 | 과목 간 AI 키워드 일관성 추적 | Orchestrator (00) | 각 보고서의 AI 키워드 사용 내역 | 키워드 현황표, 과목 간 연결고리 분석, 키워드 추천 | `reports/keyword_tracker.yaml`에 누적 기록 |

### 4.2 상세 기능 명세

#### FR1: 주제 제안

- **담당**: Topic Scout (01)
- **입력**: 과목명, 학기/단원 범위, 실험 여부
- **출력**: 주제 5개 이상 (각 주제별 AI 연결 포인트, 난이도, 예상 소요 시간 포함)
- **제약**: 교육과정 범위 준수, 이전 보고서에서 사용한 AI 키워드와의 중복/연계 고려

#### FR2: 참고문헌 및 자료 조사

- **담당**: Research Agent (02)
- **입력**: 선택된 주제, 과목 범위
- **출력**: 참고문헌 목록 (서지정보 포함), 이론적 배경 요약, 핵심 개념 정리
- ⛔ **절대 규칙**: 가짜 문헌 절대 금지. 실제 존재 여부 확인 가능한 자료만 포함

#### FR3: 실험 설계

- **담당**: Experiment Designer (03)
- **입력**: 주제, 이론 배경, 사용 가능한 장비/재료 정보
- **출력**: 탐구 질문, 가설, 변인 설정(독립/종속/통제), 준비물, 실험 절차, 안전 유의사항
- **제약**: 학교 실험실 장비로 실행 가능해야 함. 위험한 시약이나 고가 장비 사용 지양

#### FR4: 보고서 뼈대 작성

- **담당**: Report Writer (04)
- **입력**: 주제, 조사 자료, 실험 설계(해당 시), 실험 결과 데이터(학생 입력)
- **출력**: 보고서 초안 (탐구 동기, 이론 배경, 탐구 방법, 결과, 결론, 참고문헌)
- **제약**: 학생 개인화 포인트를 `[본인 경험으로 채우기]` 형태로 명시. AI 진로 연결은 1~2문장으로 자연스럽게

#### FR5: AI스러움 검토 및 수정 가이드

- **담당**: Voice Coach (05)
- **입력**: 보고서 초안
- **출력**: AI스러운 표현 목록, 각 표현에 대한 수정 제안, 전체 어투 평가
- **검출 대상**: 과도하게 정돈된 문장, 불필요한 접속부사 남용, 고등학생답지 않은 어휘, 감정 없는 서술 등

#### FR6: 주제 난이도 및 임팩트 조율

- **담당**: Topic Advisor (06)
- **입력**: 제안된 주제 목록 또는 선택된 주제
- **출력**: 난이도 평가, 임팩트 분석, 주제 변형/심화/축소 제안
- **활용 시점**: 주제 선택 단계에서 학생이 추가 상담을 원할 때

#### FR7: 면접 예상 질문 및 답변 전략

- **담당**: Interview Prep (07)
- **입력**: 완성된 보고서, 지원 프로그램/대회 정보
- **출력**: 예상 질문 목록, 각 질문별 답변 전략, 꼬리 질문 대비 포인트
- **제약**: 답변은 학생이 실제 경험한 실험/탐구 내용 기반이어야 함

#### FR8: 과목 간 AI 키워드 일관성 추적

- **담당**: Orchestrator (00)
- **입력**: 각 보고서의 AI 키워드 사용 내역
- **출력**: 키워드 사용 현황표, 과목 간 연결고리 분석, 새 보고서 작성 시 키워드 추천
- **저장**: `reports/keyword_tracker.yaml`에 누적 기록

## 5. 비기능 요구사항 (Non-Functional Requirements)

| NFR# | 요구사항 | 기준 | 구현 방법 |
|------|---------|------|-----------|
| NFR1 | **AI 생성 비감지성** | 교사가 읽었을 때 학생이 직접 작성한 것으로 인식 | Voice Coach 최종 검토 + 학생 개인화 포인트 필수 포함 |
| NFR2 | **참고문헌 실재성** | 도서명, 저자, 출판사, 출판연도 검증 가능 | Research Agent가 출처 명확히 기록 + 접근 경로 안내 |
| NFR3 | **교육과정 범위 준수** | 해당 학기 단원의 개념·용어 사용 | `config/profile.yaml`에 과목별 단원 범위 설정, 각 에이전트 참조 |
| NFR4 | **이해 수준 적절성** | 학생이 면접에서 보고서 내용을 자신의 말로 설명 가능 | 전문 용어 설명 병기, 대학 수준 수식/이론 지양 |

### 5.1 NFR 상세

**NFR1 — AI 생성 비감지성**
- ⛔ 최종 보고서가 AI에 의해 생성된 느낌을 주면 안 된다
- 검증: AI스러운 표현 체크리스트 기반 자가 검토

**NFR2 — 참고문헌 실재성**
- ⛔ 가짜 논문, 존재하지 않는 도서, 허위 저자명 사용 절대 금지
- 학생에게 직접 확인할 수 있는 접근 경로(도서관, 온라인 DB 등)를 안내

**NFR3 — 교육과정 범위 준수**
- AI 진로 연결 부분은 교과 범위를 약간 확장할 수 있으나, 핵심 탐구 내용은 범위 내로 제한

**NFR4 — 이해 수준 적절성**
- 참고: 학생의 물리 수준은 역학 기본 (비역학 미학습)

## 9. 제약 조건 및 가정 (Constraints & Assumptions)

### 9.1 절대 규칙 (Absolute Rules)

| ⛔ | 규칙 | 설명 |
|----|------|------|
| ⛔ | **가짜 문헌 금지** | 존재하지 않는 논문, 도서, 허위 저자명 사용 절대 금지 |
| ⛔ | **AI 생성 티 금지** | 교사 또는 AI 감지 도구에 의해 AI 생성물로 판별되면 안 됨 |
| ⛔ | **교육과정 이탈 금지** | 핵심 탐구 내용은 2022 개정 교육과정 범위 내로 제한 |
| ⛔ | **학생 개입 생략 금지** | 모든 보고서에 학생 본인의 경험, 관찰, 소감이 반드시 포함 |

### 9.2 제약 조건

| # | 제약 조건 | 설명 |
|---|-----------|------|
| C1 | **실험 환경** | 학교 실험실 장비와 일반적인 시약만 사용 가능. 고가 장비나 위험 시약은 불가 |
| C2 | **교육과정** | 2022 개정 교육과정 범위 내에서만 주제 선정 및 내용 작성 |
| C3 | **AI 감지 회피** | 교사 또는 AI 감지 도구에 의해 AI 생성물로 판별되면 안 됨 |
| C4 | **학생 수준** | 물리는 역학 기본만 학습. 비역학 분야(전자기, 열역학 심화 등)는 이해 범위 밖 |
| C5 | **참고문헌** | 실제 존재하는 자료만 사용. 학생이 접근 가능한 자료 우선 (학교 도서관, 공공 DB) |
| C6 | **학생 개입 필수** | 모든 보고서에 학생 본인의 경험, 관찰, 소감이 반드시 포함되어야 함 |

### 9.3 가정

| # | 가정 | 설명 |
|---|------|------|
| A1 | **프로필 최신성** | `config/profile.yaml`의 과목별 단원 범위가 학생에 의해 최신 상태로 유지된다 |
| A2 | **실험 수행** | 실험이 포함된 보고서의 경우, 학생이 직접 실험을 수행하고 결과 데이터를 입력한다 |
| A3 | **최종 다듬기** | Voice Coach의 가이드를 참고하여 학생이 최종적으로 본인 어투로 다듬는다 |
| A4 | **도구 접근성** | 학생이 Claude 기반 에이전트 시스템에 접근할 수 있는 환경이 갖춰져 있다 |
| A5 | **단일 사용자** | 현재 시스템은 1명의 학생을 위한 개인화 시스템이다 |
