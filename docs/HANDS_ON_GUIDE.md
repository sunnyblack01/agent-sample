## 0. 환경 설정 (처음 한 번만)

이 섹션은 처음 한 번만 하면 됩니다. 이미 설치가 끝났다면 [1. 빠른 시작](#1-빠른-시작-quick-start)으로 넘어가세요.

---

### 0.1 VS Code 설치

VS Code는 코드와 파일을 편집하고, 터미널에서 Claude Code를 실행하기 위한 프로그램입니다.

1. https://code.visualstudio.com 에 접속합니다.
2. 본인 운영체제에 맞는 설치 파일을 다운로드합니다.
   - **macOS**: `.dmg` 파일을 실행하고, 나타난 VS Code 아이콘을 Applications 폴더로 드래그합니다.
   - **Windows**: `.exe` 파일을 실행하고, 설치 마법사의 안내를 따라 "다음"을 눌러 설치를 완료합니다.
3. 설치가 끝나면 VS Code를 실행해봅니다.

> ✅ **확인 방법**: VS Code를 열었을 때 "Welcome" 탭이 보이면 성공입니다.

---

### 0.2 Node.js 설치

Node.js는 Claude Code CLI를 실행하기 위해 반드시 필요한 프로그램입니다.

1. https://nodejs.org 에 접속합니다.
2. **LTS** 버전(왼쪽 초록색 버튼)을 다운로드합니다. (최신 버전이 아니라 LTS를 받으세요)
3. 다운로드된 설치 파일을 실행하고, 안내를 따라 설치를 완료합니다.
4. 설치 후 터미널을 열고 아래 명령어를 입력합니다:

```bash
node --version
```

> ✅ **확인 방법**: `v20.x.x` 또는 `v22.x.x` 같은 버전 번호가 나오면 성공입니다.
> 만약 `command not found`가 뜨면 터미널을 완전히 닫고 다시 열어보세요.

---

### 0.3 Claude Code CLI 설치

Claude Code CLI는 터미널에서 Claude와 대화하며 보고서를 만들 수 있게 해주는 도구입니다.

1. 터미널을 엽니다.
   - **macOS**: Spotlight(Cmd + Space)에서 "터미널" 검색 후 실행
   - **Windows**: 시작 메뉴에서 "명령 프롬프트" 또는 "PowerShell" 검색 후 실행
2. 아래 명령어를 입력합니다:

```bash
npm install -g @anthropic-ai/claude-code
```

3. 설치가 끝나면 아래 명령어로 확인합니다:

```bash
claude --version
```

4. 처음 실행할 때는 API 키 입력 또는 로그인이 필요합니다. 화면의 안내를 따라 진행하세요.

> ✅ **확인 방법**: `claude --version` 입력 시 버전 번호가 나오면 성공입니다.
> 만약 권한 오류(`EACCES`)가 뜨면 FAQ의 [설치 중 권한 오류가 나요](#q-npm-install-할-때-권한-오류eacces가-나요) 항목을 참고하세요.

---

### 0.4 VS Code에서 Claude Code 사용하기

VS Code 안에서 바로 Claude Code를 사용할 수 있습니다.

**방법 1: VS Code 내장 터미널 사용**

1. VS Code를 엽니다.
2. 터미널을 엽니다: 키보드에서 `` Ctrl+` `` (백틱)을 누르거나, 상단 메뉴에서 Terminal > New Terminal을 클릭합니다.
3. 열린 터미널에서 `claude`를 입력하면 Claude Code가 시작됩니다.

**방법 2: VS Code Extension 설치**

1. VS Code 왼쪽 사이드바에서 Extensions 아이콘(네모 4개 모양)을 클릭합니다.
2. 검색창에 "Claude Code"를 입력합니다.
3. 나타난 확장 프로그램에서 "Install"을 클릭합니다.

> ✅ **확인 방법**: 터미널에서 `claude`를 입력했을 때 Claude Code가 실행되면 성공입니다.

### 0.5 프로젝트 가져오기

이 시스템의 파일들을 내 컴퓨터로 가져와야 합니다.

1. 터미널을 열고 아래 명령어를 순서대로 입력합니다:

```bash
git clone https://github.com/sunnyblack01/agent-sample.git
cd agent-sample
```

2. VS Code에서 이 폴더를 엽니다: 상단 메뉴 File > Open Folder에서 `agent-sample` 폴더를 선택합니다.
3. VS Code 터미널을 열고(`` Ctrl+` ``) `claude`를 입력하여 실행합니다.

> ✅ **확인 방법**: VS Code 탐색기(왼쪽 패널)에 `config/`, `docs/`, `reports/` 같은 폴더가 보이면 성공입니다.
> `git`이 없다는 오류가 나오면 FAQ의 [git이 설치되어 있지 않다고 나와요](#q-git이-설치되어-있지-않다고-나와요) 항목을 참고하세요.

### 0.6 프로필 설정

시스템이 나에게 맞는 주제와 보고서를 만들려면, 내 정보를 입력해야 합니다.

1. VS Code에서 `config/profile.yaml` 파일을 엽니다.
2. 본인 정보를 입력합니다:
   - **학년**: `고1`, `고2`, `고3` 중 선택
   - **진로**: 본인 희망 진로 (예: `AI/인공지능 학과`, `컴퓨터공학과`)
   - **과목 범위**: 이번 학기에 배우는 단원명 입력

자세한 작성법은 아래 [1. 빠른 시작](#1-빠른-시작-quick-start)의 "내 정보 입력하기"를 참고하세요.

> ✅ **확인 방법**: `profile.yaml` 파일에 본인 학년, 진로, 과목 단원이 채워져 있으면 성공입니다.

## 1. 빠른 시작 (Quick Start)

### 1-1. 내 정보 입력하기

시스템이 나에게 맞는 주제와 보고서를 만들려면, 먼저 `config/profile.yaml` 파일에 내 정보를 넣어야 합니다.

파일을 열면 이렇게 생겼습니다:

```yaml
student:
  grade: 고2
  career_goal: AI/인공지능 학과
  physics_level: 역학 기본 (비역학 미학습)
```

여기서 바꿀 부분:
- `grade`: 본인 학년으로 수정 (예: `고1`, `고2`, `고3`)
- `career_goal`: 본인 희망 진로로 수정 (예: `AI/인공지능 학과`, `컴퓨터공학과`)
- `physics_level`: 물리 학습 수준 (실험 설계할 때 참고됨)

### 1-2. 과목별 단원 범위 입력하기

같은 파일 아래쪽에 과목별 범위가 있습니다:

```yaml
subjects:
  화학:
    curriculum: 2022 개정 교육과정
    semester: 1학기
    units: []  # 여기에 단원명을 넣으세요
  물리:
    units: []
  수학:
    units: []
  생물:
    units: []
```

`units: []` 부분을 이렇게 채워주세요:

```yaml
  화학:
    curriculum: 2022 개정 교육과정
    semester: 1학기
    units:
      - "화학 결합과 분자의 세계"
      - "화학 반응과 에너지"
```

이렇게 단원명을 넣어야 시스템이 교육과정 범위를 벗어나지 않는 주제를 추천해줍니다.

> 팁: 교과서 목차를 보고 이번 학기에 배우는 단원명을 그대로 적으면 됩니다.

### 1-3. 첫 보고서 시작하기

프로필 설정이 끝났으면, 아래처럼 입력하면 바로 시작됩니다:

```
보고서 시작: 화학, 결합 단원, 실험 있음
```

이렇게 3가지만 알려주면 됩니다:
1. **과목** - 어떤 과목인지
2. **범위** - 어떤 단원/학기인지
3. **실험 여부** - 실험이 있는 수행평가인지, 없는지

## 2. 사용 시나리오별 가이드

### 시나리오 A: 실험이 포함된 수행평가 보고서

가장 기본적인 사용법입니다. 7단계를 순서대로 따라가면 됩니다.

---

#### Step 1: 시작하기

아래처럼 입력합니다.

```
보고서 시작: 화학, 결합 단원, 실험 있음
```

그러면 시스템이 프로필을 읽고, Topic Scout 에이전트가 주제를 찾기 시작합니다.

---

#### Step 2: 주제 5개 중 하나 선택

Topic Scout가 주제 5개를 제안합니다. 각 주제마다 이런 정보가 붙어있습니다:
- 주제명
- 어떤 교과 개념이 들어가는지
- AI랑 어떻게 연결되는지
- 난이도 (상/중/하)

**내가 할 일**: 마음에 드는 주제를 골라서 말하기

```
예시 입력: "2번 주제로 할게요"
```

주제가 고민되면 이렇게 말해도 됩니다:

```
"2번이랑 4번 중에 고민되는데, 뭐가 더 나을까?"
"3번이 끌리는데 너무 어렵지 않을까?"
```

시스템이 Topic Advisor 에이전트를 불러서 함께 비교해줍니다.
(자세한 내용은 아래 [3. 주제 고민이 될 때](#3-주제-고민이-될-때) 참고)

---

#### Step 3: 자료 조사 결과 확인

Research Agent가 선택한 주제에 대해 조사해옵니다:
- 관련 이론 배경
- 참고할 수 있는 책, 논문, 자료
- AI 연결 포인트

**내가 할 일**: 결과를 읽어보고 확인

```
예시 입력: "확인했어요, 다음 단계로 가주세요"
```

혹시 빠진 내용이 있으면:

```
"여기에 아레니우스 방정식 관련 내용도 추가해줘"
"참고 자료 중에 책도 하나 넣어줘"
```

---

#### Step 4: 실험 설계서 확인 + 실험 수행

Experiment Designer가 실험 설계서를 만들어줍니다:
- 탐구 질문 (예: "~하면 ~할까?")
- 가설
- 독립변인 / 종속변인 / 통제변인
- 준비물 목록
- 실험 절차 (순서대로)

**내가 할 일**: 설계서를 확인하고, 실제로 실험을 수행합니다.

```
예시 입력: "실험 설계 확인했어요. 실험하고 올게요!"
```

실험 준비물이 학교에 없으면:

```
"과망간산칼륨이 없는데 다른 걸로 바꿀 수 있어?"
"핫플레이트 대신 알코올 램프로 해도 돼?"
```

---

#### Step 5: 실험 결과 데이터 입력

실험을 마치고 돌아와서, 결과 데이터를 입력합니다.

```
예시 입력:

실험 결과야.

| 농도(M) | 1차(초) | 2차(초) | 3차(초) | 평균(초) |
|---------|---------|---------|---------|----------|
| 0.01    | 45.2    | 43.8    | 44.5    | 44.5     |
| 0.02    | 28.1    | 27.5    | 29.0    | 28.2     |
| 0.05    | 12.3    | 11.8    | 12.0    | 12.0     |

관찰: 농도가 높을수록 색 변화가 빨리 일어남. 0.05M에서는 거의 바로 변함.
```

표 형태가 아니라 글로 써도 됩니다:

```
"1차 실험: 45.2초, 2차: 43.8초, 3차: 44.5초였어.
색이 갑자기 확 변해서 좀 놀랐음."
```

> 팁: 실험하면서 느낀 점, 예상과 달랐던 것, 실수한 것도 같이 적어주면 보고서가 훨씬 자연스러워집니다.

---

#### Step 6: 보고서 뼈대 확인 + [✍️] 부분 채우기

Report Writer가 보고서 초안을 만들어줍니다. 초안에는 `[✍️]` 표시가 있는 부분이 있습니다.

```
## 1. 탐구 동기
수업 시간에 반응 속도에 대해 배우면서, 같은 반응이라도
조건에 따라 속도가 달라진다는 걸 알게 되었다.

[✍️ 본인 경험으로 채우기 — 예: 수업에서 본 영상, 궁금했던 점 등]
```

**내가 할 일**: `[✍️]` 표시가 있는 곳을 본인의 말로 직접 채워넣기

이 부분이 가장 중요합니다. 여기를 채우지 않으면 보고서가 AI가 쓴 것처럼 보입니다.

```
예시 입력: "[✍️] 부분 다 채웠어요. 다음 단계로 가주세요"
```

채운 내용을 함께 보내도 됩니다:

```
"탐구 동기에 이렇게 썼어:
'수업 시간에 시계반응 영상을 봤는데, 갑자기 색이 확 바뀌는 게
마술 같아서 왜 그런지 궁금했다. 집에서 영상을 다시 찾아보다가
농도를 바꾸면 시간이 달라진다는 댓글을 보고 직접 해보고 싶었다.'
이거 괜찮아?"
```

---

#### Step 7: Voice Coach 검토 + 수정

Voice Coach가 보고서를 읽고 AI스러운 부분을 찾아냅니다.

검토 결과는 이런 형태입니다:
- "체계적으로"라는 표현이 AI 느낌 → "농도도 바꿔보고 온도도 바꿔보면" 식으로 풀어쓰기
- "~이다" 종결이 너무 많음 → 중간에 "~인 것 같다", "~해서 신기했다" 섞기
- 감정/경험이 전혀 없음 → "처음에는 왜 이런지 몰랐는데" 같은 문장 추가

**내가 할 일**: 검토 결과를 보고 보고서를 직접 다듬기

```
예시 입력: "Voice Coach 가이드대로 수정했어요. 다시 봐주세요"
```

수정이 어려우면:

```
"3번 지적은 어떻게 고쳐야 할지 모르겠어. 좀 더 알려줘"
```

> Voice Coach는 직접 고쳐서 완성본을 주지 않습니다. 방향만 알려주고, 본인이 직접 쓰도록 도와줍니다. 이게 핵심입니다.

---

### 시나리오 B: 실험 없는 보고서

실험이 없는 수행평가(문헌 조사 보고서, 분석 보고서 등)는 위 과정에서 **Step 4(실험 설계)와 Step 5(실험 결과 입력)를 건너뜁니다**.

```
보고서 시작: 수학, 함수 단원, 실험 없음
```

진행 순서:
1. Step 1: 시작하기
2. Step 2: 주제 선택
3. Step 3: 자료 조사 확인
4. Step 6: 보고서 뼈대 확인 + [✍️] 채우기
5. Step 7: Voice Coach 검토 + 수정

나머지는 시나리오 A와 동일합니다.

---

### 시나리오 C: 프로그램/대회 지원 + 면접 준비

수행평가가 아니라 프로그램 지원이나 대회 출품용 보고서라면, 보고서 작성 후에 Interview Prep 에이전트를 추가로 사용합니다.

#### 보고서 작성

시나리오 A 또는 B와 동일하게 보고서를 먼저 완성합니다.

#### 면접 준비 시작

보고서가 끝나면 이렇게 입력합니다:

```
"면접 준비 시작해줘. 이건 화학실험 프로그램 지원용이야."
```

또는 더 구체적으로:

```
"면접 준비 해줘. 이 보고서로 영재원 지원하려고 해.
면접 시간은 5분이고, 교수님 2명이 질문해."
```

#### 면접 시뮬레이션 활용

Interview Prep이 세 종류의 예상 질문을 만들어줍니다:

1. **보고서 기반 질문** (5-7개) - 면접관이 보고서를 읽고 물어볼 질문
2. **진로 연결 질문** (3-5개) - "왜 이 주제를 선택했나요?" 같은 질문
3. **돌발/심화 질문** (3-5개) - 보고서 범위를 넘는 질문

각 질문마다 답변 포인트와 주의사항이 함께 나옵니다.

**면접 연습하는 법**:

```
"시뮬레이션 해보고 싶어. 면접관처럼 질문해줘"
```

그러면 실제 면접처럼 질문이 하나씩 나오고, 내가 답변하면 피드백을 받을 수 있습니다.

```
"이 질문에 이렇게 답하려는데 괜찮아?
'처음에 시계반응 영상을 보고 신기해서 시작했는데,
실험하다 보니까 데이터에서 규칙을 찾는 게 재미있었습니다.'"
```

> 주의: 면접 답변도 너무 완벽하면 의심받습니다. "모르겠지만 ~라고 추측합니다" 같은 솔직한 답변이 오히려 좋은 인상을 줍니다.

## 3. 주제 고민이 될 때

Topic Scout가 주제 5개를 제안했는데 뭘 골라야 할지 모르겠다면, Topic Advisor를 활용하세요.

### 기본 사용법

```
"주제 고민돼. 2번이랑 4번 비교해줘"
```

Topic Advisor가 난이도, 임팩트, AI 연결 자연스러움, 실현 가능성 등을 기준으로 비교해줍니다.

### 피드백 방법

솔직하게 느낌을 말하면 됩니다:

| 이렇게 느낄 때 | 이렇게 말하면 됩니다 |
|---------------|---------------------|
| 주제가 너무 쉬워 보일 때 | `"이거 너무 쉽지 않아? 좀 더 어려운 거 없어?"` |
| 주제가 너무 어려워 보일 때 | `"이건 내 수준에서 할 수 있는 건지 모르겠어"` |
| 임팩트가 없어 보일 때 | `"이걸로 하면 보고서가 별로 안 특별해 보일 것 같아"` |
| AI 연결이 억지 같을 때 | `"AI랑 연결하는 게 좀 억지스럽지 않아?"` |
| 실험이 현실적으로 어려울 때 | `"우리 학교에 이 장비가 없는데..."` |
| 두 주제를 합치고 싶을 때 | `"1번 실험에 3번의 AI 연결을 붙이면 어때?"` |

Topic Advisor는 피드백을 반영해서 기존 주제를 변형하거나 새로운 조합을 제안합니다. 최종 선택은 항상 본인이 합니다.

## 4. 보고서 다듬기 팁

### Voice Coach 검토 결과 활용법

Voice Coach가 지적한 부분은 보통 이런 형태입니다:

```
원문: "이 요인들을 체계적으로 바꿔 보면"
문제: "체계적으로"는 AI가 좋아하는 표현
수정 방향: "농도도 바꿔보고 온도도 바꿔보면" 식으로 구체적으로 풀기
```

이걸 보고 본인이 직접 문장을 고치면 됩니다. Voice Coach는 답을 주지 않고 방향만 알려주기 때문에, 본인의 말투가 자연스럽게 담기게 됩니다.

수정한 뒤 다시 검토를 요청하세요:

```
"수정했어. 다시 검토해줘"
```

한 번에 완벽할 필요 없습니다. 2-3번 반복하면 충분합니다.

### AI 티가 나지 않게 쓰는 핵심 원칙 5가지

#### 원칙 1: "체계적으로", "결국", "구조" 같은 단어 줄이기

AI는 이런 단어를 매우 좋아합니다. 보고서에서 이 단어들이 보이면 다른 말로 바꾸세요.

| 줄여야 할 표현 | 바꿀 수 있는 표현 |
|---------------|-----------------|
| 체계적으로 분석하면 | 하나씩 따져보면 |
| 결국 | 그래서 / 해보니까 |
| 구조적으로 | (빼거나) 이렇게 생겨서 |
| 효율적으로 | 빠르게 / 덜 귀찮게 |
| 유의미한 결과 | 확실히 차이가 남 |
| 도출할 수 있다 | 알 수 있었다 |

#### 원칙 2: 본인 경험/에피소드 넣기

AI는 경험을 지어내지 못합니다. 본인만의 구체적인 경험을 넣으면 자연스러워집니다.

- "수업 시간에 선생님이 보여준 영상에서 ~"
- "작년에 과학 동아리에서 비슷한 실험을 했는데 ~"
- "유튜브에서 우연히 본 영상이 계기가 됐다"

#### 원칙 3: 실패나 혼란 드러내기

AI가 쓴 보고서는 너무 매끄럽습니다. 실제 탐구에서는 헤매는 과정이 있기 마련입니다.

- "처음에는 왜 이런 결과가 나오는지 전혀 모르겠었다"
- "두 번째 실험에서 실수로 농도를 잘못 맞춰서 다시 했다"
- "가설과 다른 결과가 나와서 당황했는데, 다시 생각해보니 ~"

#### 원칙 4: 짧은 문장으로 끊어쓰기

AI는 한 문장 안에 많은 정보를 넣으려 합니다. 짧게 끊어주세요.

변경 전:
> "아이오딘 시계반응의 반응 속도에 영향을 미치는 요인을 탐구하기 위해 농도와 온도를 변인으로 설정하고 체계적인 실험을 설계하였다."

변경 후:
> "시계반응이 빨라지는 조건이 뭔지 알고 싶었다. 그래서 농도랑 온도를 바꿔가면서 실험해봤다."

#### 원칙 5: "~이다" 종결 남발 금지

보고서 전체가 "~이다"로 끝나면 교과서 같은 느낌이 납니다. 중간중간 다른 종결을 섞어주세요.

- "~인 것 같다"
- "~해서 신기했다"
- "~한 결과를 얻었다"
- "~는지 궁금했다"
- "~라는 걸 알게 됐다"

## 5. FAQ

### Q: Node.js가 설치되지 않아요 / `node` 명령어를 찾을 수 없다고 나와요

**A:** 설치 후 터미널을 닫고 새로 열어보세요. 그래도 안 되면 컴퓨터를 재시작해보세요. macOS에서 Homebrew를 사용한다면 `brew install node`로도 설치할 수 있습니다. Windows에서는 설치 마법사에서 "Add to PATH" 옵션이 체크되어 있는지 확인하세요.

---

### Q: `npm install` 할 때 권한 오류(EACCES)가 나요

**A:** macOS/Linux에서 흔한 문제입니다. 아래 두 가지 방법 중 하나를 시도하세요:
1. 명령어 앞에 `sudo`를 붙이기: `sudo npm install -g @anthropic-ai/claude-code` (비밀번호 입력 필요)
2. npm 기본 디렉토리 권한 변경: `sudo chown -R $(whoami) /usr/local/lib/node_modules`

Windows에서는 명령 프롬프트를 "관리자 권한으로 실행"한 뒤 다시 시도하세요.

---

### Q: `git`이 설치되어 있지 않다고 나와요

**A:** Git이 없으면 프로젝트를 clone할 수 없습니다.
- **macOS**: 터미널에서 `xcode-select --install`을 입력하면 Git이 함께 설치됩니다.
- **Windows**: https://git-scm.com 에서 다운로드하여 설치하세요. 설치 후 터미널을 다시 열어야 합니다.

---

### Q: `claude` 명령어가 안 돼요

**A:** 몇 가지 확인할 사항이 있습니다:
1. Node.js가 정상 설치되었는지 확인: `node --version`
2. Claude Code가 정상 설치되었는지 확인: `npm list -g @anthropic-ai/claude-code`
3. 위 두 가지 모두 정상이면 터미널을 닫고 다시 열어보세요.
4. 그래도 안 되면 `npx @anthropic-ai/claude-code`로 실행해보세요.

---

### Q: 다른 과목에서도 쓸 수 있어요?

**A:** 네, 과목과 범위만 바꾸면 됩니다. `config/profile.yaml`에서 해당 과목의 `units`를 채우고, "보고서 시작: 생물, 세포 단원, 실험 있음" 식으로 입력하면 됩니다. 화학, 물리, 수학, 생물 모두 같은 방식으로 사용할 수 있습니다.

---

### Q: 보고서 전체를 그대로 내도 되나요?

**A:** 안 됩니다. 시스템이 만들어주는 건 "뼈대"입니다. 반드시 아래 두 가지를 해야 합니다:
1. `[✍️]` 표시된 부분을 본인 경험과 생각으로 채우기
2. Voice Coach 가이드를 참고해서 전체적으로 본인 말투로 다듬기

이 두 단계를 건너뛰면 AI가 쓴 것처럼 보여서 오히려 감점될 수 있습니다.

---

### Q: 이전 보고서에서 쓴 AI 키워드를 어떻게 확인해요?

**A:** `reports/keyword_tracker.yaml` 파일을 열어보면 과목별로 사용한 AI 연결 키워드(분류, 예측, 패턴 인식 등)가 기록되어 있습니다. 다음 보고서를 쓸 때 같은 키워드를 반복하지 않거나, 다른 각도로 연결하는 데 참고할 수 있습니다.

---

### Q: 중간에 멈추고 나중에 이어서 할 수 있어요?

**A:** 네. 각 단계의 결과는 `reports/{과목}_{주제}/` 폴더에 자동으로 저장됩니다. 나중에 돌아와서 "이어서 진행해줘"라고 하면, 저장된 파일을 읽고 다음 단계부터 계속됩니다.

---

### Q: 주제를 선택한 후에 바꿀 수 있어요?

**A:** 자료 조사 전이라면 "주제 바꿀래. 다른 거로 할게"라고 하면 됩니다. 이미 자료 조사나 실험 설계까지 진행했다면 처음부터 다시 시작해야 하니, 주제 선택 단계에서 충분히 고민하는 게 좋습니다. Topic Advisor를 적극 활용하세요.

---

### Q: 실험을 했는데 가설과 결과가 다르면 어떡해요?

**A:** 전혀 문제 없습니다. 오히려 좋습니다. 가설과 다른 결과가 나왔다는 것 자체가 좋은 탐구 내용이 됩니다. 결과 데이터를 그대로 입력하면, Report Writer가 "가설과 달랐던 이유"를 분석하는 방향으로 보고서를 작성해줍니다. 결과를 조작하거나 가설에 맞춰 바꾸면 절대 안 됩니다.

---

### Q: 참고 문헌이 진짜 있는 건지 어떻게 확인해요?

**A:** Research Agent가 제시하는 참고 문헌은 실제 존재하는 것만 사용하도록 설정되어 있습니다. 그래도 제출 전에 책 제목이나 논문 제목을 검색해서 한 번 더 확인하는 게 안전합니다. 가짜 출처가 들어가면 큰 감점 요인이 됩니다.

---

### Q: 면접 준비는 꼭 해야 하나요?

**A:** 일반 수행평가라면 하지 않아도 됩니다. 프로그램 지원, 영재원, 대회처럼 면접이나 발표가 있는 경우에만 사용하면 됩니다. 보고서 완성 후 "면접 준비 시작해줘"라고 입력하세요.
