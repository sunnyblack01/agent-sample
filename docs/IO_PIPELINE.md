## 1. 파이프라인 개요

> 본 문서는 7개 에이전트 + 오케스트레이터로 구성된 수행평가 보고서 시스템의 데이터 흐름, 각 에이전트별 입출력 명세, 저장 경로, 학생 개입 포인트를 정리한다.

### 1.1 전체 흐름도 — 실험 포함 워크플로우

```
┌───────────────────────────────────────────────────────────────────────┐
│                      Orchestrator (Agent 00)                         │
│          config/profile.yaml 로드 / keyword_tracker.yaml 참조          │
└───────────┬───────────────────────────────────────────────────────────┘
            │
            ▼
  ┌───────────────────┐
  │    학생 입력        │   과목, 범위, 실험 여부
  └─────────┬─────────┘
            │
            ▼
  ┌───────────────────┐
  │  Agent 01          │
  │  Topic Scout       │
  └─────────┬─────────┘
            │  주제 후보 5개+
            ▼
  ┌───────────────────┐
  │  Agent 06          │ ◀── 학생 반응/선호
  │  Topic Advisor     │
  └─────────┬─────────┘
            │  다면평가 + 추천
            ▼
  ┌───────────────────┐
  │  학생 주제 선택     │
  └─────────┬─────────┘
            │
            ▼
  ┌───────────────────┐
  │  Agent 02          │
  │  Research Agent    │
  └─────────┬─────────┘
            │  이론 + 참고문헌
            ▼
  ┌───────────────────┐
  │  Agent 03          │
  │  Experiment        │
  │  Designer          │
  └─────────┬─────────┘
            │  실험 설계
            ▼
  ┌───────────────────┐
  │  학생 실험 수행     │
  │  결과 데이터 입력   │
  └─────────┬─────────┘
            │
            ▼
  ┌───────────────────┐
  │  Agent 04          │
  │  Report Writer     │
  └─────────┬─────────┘
            │  보고서 뼈대 ([✍️] 포함)
            ▼
  ┌───────────────────┐
  │  Agent 05          │
  │  Voice Coach       │
  └─────────┬─────────┘
            │  수정 가이드
            ▼
  ┌───────────────────┐
  │  학생 최종 다듬기   │
  └─────────┬─────────┘
            │
            ▼
  ┌───────────────────┐
  │  Agent 07          │
  │  Interview Prep    │
  └─────────┬─────────┘
            │  예상질문 + 시뮬레이션
            ▼
  ┌───────────────────┐
  │   최종 산출물       │
  └───────────────────┘
```

### 1.2 전체 흐름도 — 실험 없는 워크플로우

```
  학생 입력
     │
     ▼
  Agent 01 → (Agent 06) → 학생 선택 → Agent 02
                                          │
                                          ▼
  Agent 07 ◀── 학생 다듬기 ◀── Agent 05 ◀── Agent 04
```

> Agent 03 (Experiment Designer)을 건너뛰고, Agent 04가 실험 섹션 없이 보고서를 구성한다.

## 2. 시나리오별 I/O 명세

### 2.1 Agent 00 — Orchestrator

| 구분 | 항목 | 설명 |
|------|------|------|
| **Input** | 학생 초기 요청 | `"보고서 시작: 화학, 1학기, 실험 있음"` 형태 |
| | `config/profile.yaml` | 학생 프로필, 진로 전략, 과목 범위, 제약 조건 |
| | `reports/keyword_tracker.yaml` | 이전 과목에서 사용한 AI 키워드 이력 |
| **Output** | 워크플로우 제어 | 각 에이전트 순차 호출 및 결과 전달 |
| | 단계별 요약 | 학생에게 현재 결과를 보여주고 확인 요청 |
| | 키워드 갱신 | `keyword_tracker.yaml` 업데이트 |

---

### 2.2 Agent 01 — Topic Scout (주제 발굴)

**Input 테이블**

| 항목 | 형식 | 상세 |
|------|------|------|
| 과목명 | `string` | 예: "화학" |
| 단원/범위 | `string` | 예: "1학기 산화-환원" |
| 실험 필수 여부 | `boolean` | `true` / `false` |
| 이전 사용 키워드 목록 | `list` | Orchestrator가 `keyword_tracker.yaml`에서 추출 |

**Output 테이블** — 주제 후보 5개 이상, 각 후보별 아래 필드 포함

| 필드 | 설명 | 예시 |
|------|------|------|
| 주제명 | 한 줄 제목 | "시계 반응 속도에 영향을 미치는 요인 탐구" |
| 핵심 개념 | 해당 과목 교과 내용 | 반응 속도, 촉매, 활성화 에너지 |
| AI 연결 고리 | 연결되는 AI 개념 | 패턴 인식, 예측 모델 |
| 실험 가능 여부 | 학교 실험실 수행 가능 여부 | O / X |
| 실험 개요 | 간단한 실험 방향 | 농도별 시계 반응 시간 측정 |
| 난이도 | 상 / 중 / 하 | 중 |
| 진로 어필 포인트 | 보고서 내 진로 연결 서술 방향 | "반응 시간 예측 = 회귀 분석" |

**저장 경로**: `reports/{과목}_{주제}/` 폴더 생성 시점 (주제 확정 후)

---

### 2.3 Agent 02 — Research Agent (자료 조사)

**Input 테이블**

| 항목 | 형식 | 상세 |
|------|------|------|
| 선택된 주제 | `string` | Agent 01 결과 중 학생이 선택한 1건 |
| 과목 및 단원 | `string` | — |
| AI 연결 키워드 | `list` | Agent 01이 제시한 AI 연결 고리 |

**Output 테이블**

| 항목 | 형식 | 상세 |
|------|------|------|
| 이론 정리 | `markdown` | 교과서 기반 핵심 개념 요약 |
| 참고 도서 2–3권 | `list` | 제목/저자/출판사/관련 챕터/추천 이유 |
| 논문/기사 2–3건 | `list` | 제목/출처/핵심 내용/인용 포인트 |
| AI 관련 자료 1–2건 | `list` | 해당 분야 AI 활용 사례 (뉴스/연구) |

**저장 경로**: `reports/{과목}_{주제}/research.md`

---

### 2.4 Agent 03 — Experiment Designer (실험 설계)

**Input 테이블**

| 항목 | 형식 | 상세 |
|------|------|------|
| 선택된 주제 | `string` | — |
| 이론 배경 | `markdown` | Agent 02 결과 (`research.md`) |
| 학교 실험실 환경 | — | 기본 장비 가정 |

**Output 테이블**

| 항목 | 형식 | 상세 |
|------|------|------|
| 탐구 질문 | `string` | "~하면 ~할까?" 형태 |
| 가설 | `string` | 예상 결과 + 이론적 근거 |
| 변인 설정 | `table` | 독립변인 / 종속변인 / 통제변인 |
| 준비물 | `list` | 시약/도구 목록 (대체재 포함) |
| 실험 절차 | `list` | 단계별 상세 절차 + 안전 주의사항 |
| 데이터 기록 양식 | `table` | 표 템플릿 (측정 단위, 반복 횟수) |
| 예측-검증 설계 | `string` | 가설 기반 예측과 검증 방법 |
| AI 관점 연결 | `string` | 데이터 수집/분석 과정과 AI 유사성 |

**저장 경로**: `reports/{과목}_{주제}/experiment_design.md`

---

### 2.5 Agent 04 — Report Writer (보고서 작성)

**Input 테이블**

| 항목 | 형식 | 상세 |
|------|------|------|
| 주제 | `string` | — |
| 이론 배경 | `markdown` | Agent 02 결과 |
| 참고 문헌 목록 | `list` | Agent 02 결과 내 문헌 리스트 |
| 실험 설계 | `markdown` | Agent 03 결과 (실험 있는 경우) |
| 실험 결과 데이터 | 학생 입력 | 학생이 직접 입력 |
| `config/profile.yaml` | `yaml` | 학생 프로필, 진로 전략 |

**Output 테이블** — 보고서 전체 뼈대 (`templates/report_template.md` 기반)

| 섹션 | 내용 | `[✍️]` 마커 |
|------|------|-------------|
| 제목 | 탐구 내용이 드러나는 구체적 제목 | — |
| 1. 탐구 동기 | 수업 연결 + AI 진로 연결 | `[✍️ 본인 경험으로 채우기]` |
| 2. 이론적 배경 | 교과 개념 + 선행 연구 인용 | — |
| 3. 탐구 방법 | 질문/가설/변인/준비물/절차 | — |
| 4. 탐구 결과 | 데이터 표/그래프 + 관찰 내용 | `[✍️ 직접 관찰한 내용 적기]` |
| 5. 결론 및 고찰 | 가설 검증 + 해석 + 한계 | `[✍️ 본인이 느낀 점 추가]` |
| 6. 참고 문헌 | APA 형식 | — |

**저장 경로**: `reports/{과목}_{주제}/report_draft.md`

### 2.6 Agent 05 — Voice Coach (어투 코치)

**Input 테이블**

| 항목 | 형식 | 상세 |
|------|------|------|
| 보고서 초안 텍스트 | `markdown` | Agent 04 결과 (`report_draft.md`) |

**Output 테이블**

| 항목 | 형식 | 상세 |
|------|------|------|
| AI 느낌 체크리스트 | `checklist` | 8개 항목 점검 (해당 문장 지적) |
| 수정 제안 | `list` | 원문 / 문제점 / 수정 방향 (방향만, 완성본 X) |
| 학생다움 추가 포인트 | `list` | 경험/감정을 넣을 위치 안내 |

**AI 느낌 체크 항목**

| # | 점검 항목 |
|---|-----------|
| 1 | 과도하게 정돈된 문장 구조 |
| 2 | 모든 문단이 똑같은 길이 |
| 3 | "~라고 할 수 있다" 반복 |
| 4 | 불필요한 접속사 남발 ("또한", "더불어", "나아가") |
| 5 | 감정/경험이 전혀 없는 건조한 톤 |
| 6 | 너무 완벽한 논리 흐름 (실수/의문 없음) |
| 7 | 이모지나 과도한 강조 |
| 8 | 교과서 문장 그대로 복붙한 느낌 |

**저장 경로**: `reports/{과목}_{주제}/voice_coach_review.md`

### 2.7 Agent 06 — Topic Advisor (주제 심화 토론)

**Input 테이블**

| 항목 | 형식 | 상세 |
|------|------|------|
| 주제 후보 목록 | `list` | Agent 01 결과 (5개+) |
| 학생 반응 | `string` | 선호/거부 이유, "너무 쉽다" 등 |
| 프로그램 맥락 | `string` | 수행평가 / 프로그램 지원 / 대회 |

**Output 테이블**

| 항목 | 형식 | 상세 |
|------|------|------|
| 주제별 다면 평가표 | `table` | 6개 기준 x 1–5점 + 근거 |
| 변형/조합 제안 | `list` | 기존 주제 변형 또는 주제 간 조합 |
| 최종 추천 | `string` | 맥락별 추천 + 선택 시 주의점 |

**다면 평가 기준**

| 기준 | 가중치 (수행평가) | 가중치 (프로그램 지원) | 가중치 (대회) |
|------|:-:|:-:|:-:|
| 난이도 적절성 | 높음 | 보통 | 보통 |
| 임팩트/차별성 | 보통 | 높음 | 높음 |
| AI 연결 자연스러움 | 높음 | 높음 | 보통 |
| 실현 가능성 | 높음 | 보통 | 보통 |
| 면접 설명력 | 보통 | 높음 | 낮음 |
| 성장 서사 | 낮음 | 높음 | 보통 |

**저장 경로**: 별도 파일 저장 없음 (대화 내에서 처리, 선택 결과는 Orchestrator가 관리)

|------|------|------|
| 제출 보고서 | `markdown` | 최종 보고서 전문 |

**저장 경로**: `reports/{과목}_{주제}/interview_prep.md`

## 3. 데이터 흐름 상세

### 3.1 파일 경로 맵

```
agent sample/
├── config/
│   └── profile.yaml                       ← [고정] 학생 프로필 및 설정
├── templates/
│   └── report_template.md                 ← [고정] 보고서 템플릿
├── reports/
│   ├── keyword_tracker.yaml               ← [누적] AI 키워드 추적 (전 과목 공유)
│   └── {과목}_{주제}/                       ← [과제별] 보고서 폴더
│       ├── research.md                    ← Agent 02 출력
│       ├── experiment_design.md           ← Agent 03 출력
│       ├── report_draft.md                ← Agent 04 출력
│       ├── voice_coach_review.md          ← Agent 05 출력
│       └── interview_prep.md              ← Agent 07 출력
└── agents/
    ├── 00_orchestrator.md                 ← [고정] 워크플로우 정의
    ├── 01_topic_scout.md                  ← [고정] 에이전트 프롬프트
    ├── ...
    └── 07_interview_prep.md               ← [고정] 에이전트 프롬프트
```

### 3.2 에이전트 간 데이터 의존성

```
┌───────────┐
│ Agent 01  │───────────────────────────┐
│ Topic     │                           │
│ Scout     │──┐                        │
```

## 4. 학생 개입 포인트

워크플로우에서 학생이 직접 행동해야 하는 지점은 총 **5곳**이다.

### 4.1 개입 흐름도

```
  ┌───────────────────────┐
  │  Agent 01              │
  │  Topic Scout           │
  └───────────┬───────────┘
              ▼
  ┌───────────────────────┐
  │ ⛔ 개입 1 [필수 입력]   │  과목, 단원/범위, 실험 여부 입력
  │                        │  형태: "보고서 시작: 화학, 1학기, 실험 있음"
  └───────────┬───────────┘
              ▼
  ┌───────────────────────┐
  │  Agent 06              │ ◀── ✅ 개입 2 [선택]: 주제에 대한 반응/선호 전달
  │  Topic Advisor         │
  └───────────┬───────────┘
              ▼
  ┌───────────────────────┐
  │ ⛔ 개입 3 [필수 선택]   │  주제 후보 중 최종 1개 선택
  └───────────┬───────────┘
              ▼
  ┌───────────────────────┐
  │  Agent 02 → Agent 03   │
  └───────────┬───────────┘
              ▼
  ┌───────────────────────┐
  │ ⛔ 개입 4 [조건부 필수] │  실험 직접 수행 후 결과 데이터 입력
  │                        │  (실험 포함 워크플로우에서만)
  └───────────┬───────────┘
              ▼
  ┌───────────────────────┐
  │  Agent 04 → Agent 05   │
  └───────────┬───────────┘
              ▼
  ┌───────────────────────┐
  │ ⛔ 개입 5 [필수 작업]   │  Voice Coach 가이드 참고하여 본인 어투로 최종 다듬기
  │                        │  [✍️] 마커 부분을 본인 경험/감정으로 채우기
  └───────────┬───────────┘
              ▼
  ┌───────────────────────┐
  │  Agent 07              │
  │  Interview Prep        │
  └───────────────────────┘
```

### 4.2 개입 포인트 상세

| # | 시점 | 유형 | 학생이 해야 할 일 | 필수 여부 |
|---|------|------|-------------------|-----------|
| 1 | 워크플로우 시작 전 | 정보 입력 | 과목명, 단원/범위, 실험 포함 여부 전달 | ⛔ 필수 |
| 2 | 주제 후보 제시 후 | 피드백 | 각 주제에 대한 느낌/선호도 전달 (Agent 06 활성화) | ✅ 선택 |
| 3 | 주제 평가 후 | 의사결정 | 최종 주제 1개 확정 | ⛔ 필수 |
| 4 | 실험 설계 후 | 실험 수행 + 데이터 입력 | 직접 실험하고 결과 데이터를 시스템에 입력 | ⛔ 조건부 필수 |
| 5 | Voice Coach 리뷰 후 | 보고서 수정 | `[✍️]` 마커 부분 채우기 + AI스러운 표현 본인 어투로 수정 | ⛔ 필수 |

> **핵심 원칙**: 시스템은 뼈대와 가이드를 제공하고, 학생 고유의 경험/어투/감정은 반드시 학생이 직접 작성한다. 이는 AI 사용 흔적을 줄이고 진정성 있는 보고서를 만들기 위한 설계 원칙이다.

## 5. 런타임 데이터 생명주기

### 5.1 파일별 생성·갱신 주기

| 파일 | 생성 시점 | 갱신 여부 | 비고 |
|------|-----------|-----------|------|

| `keyword_tracker.yaml` | 최초 실행 시 | 매 보고서 완료 시 추가 | 과목 간 키워드 일관성 관리 |

| `experiment_design.md` | Agent 03 완료 시 | 학생 피드백 반영 가능 | 실험 없는 경우 미생성 |
| `report_draft.md` | Agent 04 완료 시 | 학생이 `[✍️]` 채운 후 갱신 | 최종 제출본 |
| `voice_coach_review.md` | Agent 05 완료 시 | 1회성 | 수정 가이드 역할 |
| `interview_prep.md` | Agent 07 완료 시 | 1회성 | 면접 연습용 |
